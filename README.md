# Лабораторная работа: очередь на C++ (вариант 11)

Этот проект — простая реализация очереди **без STL-контейнеров**.

## По заданию нужно найти:
> **Количество элементов очереди, которые больше среднего гармонического этой очереди.**

В программе это выполняется в пункте меню **4**.

---

## Что реализовано

### 1) Базовый класс `Queue`

Файлы: `Queue.h`, `Queue.cpp`

Класс умеет:
- `enqueue(int)` — добавить элемент в очередь;
- `dequeue(int&)` — извлечь первый (самый старый) элемент;
- `print()` — вывести очередь;
- `createCopy()` — сделать копию очереди;
- `merge(q1, q2)` — слить две очереди в новую;
- `getHarmonicMean(double&)` — вычислить среднее гармоническое;
- `countElementsGreaterThan(double)` — посчитать элементы больше указанного значения.

Внутри используется собственный узел:
- `data` — число;
- `prev` — ссылка на предыдущий элемент.

### 2) Три варианта наследования

- `Queue1Public` (`public`)
- `Queue1Protected` (`protected`)
- `Queue1Private` (`private`)

Файлы:
- `Queue1Public.h/.cpp`
- `Queue1Protected.h/.cpp`
- `Queue1Private.h/.cpp`

Во всех трех классах есть методы для вычисления варианта:
- `findRequiredValue()`
- `printRequiredValue()`

### 3) Консольное меню

Файл: `main.cpp`

Есть 2 уровня меню:
1. Выбор режима наследования (`public`, `protected`, `private`).
2. Работа с очередью:
   - 1 — добавить элемент
   - 2 — извлечь элемент
   - 3 — вывести очередь
   - 4 — вычислить требуемое значение
   - 5 — создать копию
   - 6 — слияние исходной очереди с копией
   - 7 — выход

---

## Формула среднего гармонического

Для чисел `x1, x2, ..., xn`:

\[
H = \frac{n}{\frac{1}{x_1} + \frac{1}{x_2} + ... + \frac{1}{x_n}}
\]

### Важные ограничения

Если очередь:
- пустая,
- или содержит `0`,

то среднее гармоническое в программе **не вычисляется** (выводится сообщение).


## Короткий пример работы

Допустим, вы добавили: `2, 4, 8`.

- Среднее гармоническое: `3.42857`
- Элементы больше него: `4` и `8`
- Ответ: `2`

## Как объяснить на защите (очень кратко)

1. Очередь реализована вручную, без STL, через динамические узлы.
2. Каждый узел хранит `int` и `prev`.
3. Есть копирование и слияние с сохранением порядка.
4. Есть 3 вида наследования (`public/protected/private`).
5. По варианту 11 считаем количество элементов больше среднего гармонического.
