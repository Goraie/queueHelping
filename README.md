# Лабораторная работа: Очередь (вариант 11)

Проект написан максимально простым стилем, без STL-контейнеров.

## Что требуется по варианту 11

Нужно найти:

**количество элементов очереди, которые больше среднего гармонического этой очереди**.

В коде это реализовано в `printRequiredValue()` / `findRequiredValue()` у классов:
- `Queue1Public`
- `Queue1Protected`
- `Queue1Private`

## Как устроена очередь

Элемент очереди (узел) хранит:
- целое число `data`,
- указатель `prev` на предыдущий элемент.

Внутри класса `Queue` хранится `tail` — указатель на последний добавленный элемент.

### Операции

- `enqueue(value)` — добавить элемент.
- `dequeue(value)` — извлечь первый (самый старый) элемент.
- `print()` — вывести очередь в порядке FIFO.
- `createCopy()` — создать копию очереди.
- `merge(q1, q2)` — сделать новую очередь: сначала `q1`, потом `q2`.
- `getHarmonicMean(mean)` — посчитать среднее гармоническое.
- `countElementsGreaterThan(value)` — посчитать элементы больше заданного числа.

---

## Важный момент про среднее гармоническое

Формула:

\[
H = \frac{n}{\sum_{i=1}^{n} \frac{1}{x_i}}
\]

Если очередь пустая или содержит `0`, в программе выводится сообщение,
что среднее гармоническое вычислить нельзя.

## Меню программы

В каждом режиме наследования есть пункты:

1. Добавление элемента
2. Извлечение элемента
3. Вывод очереди
4. Вычисление требуемого значения (вариант 11)
5. Создание копии очереди
6. Слияние оригинальной очереди с копией
7. Выход
