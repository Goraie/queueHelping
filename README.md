# Лабораторная работа №1: Очередь и наследование

Этот проект реализует лабораторное задание по ООП на C++:
- базовый класс **`Queue`** (очередь на динамической связной структуре),
- дочерний класс **`Queue1`** в трех вариантах наследования:
  - `public` (`Queue1Public`),
  - `protected` (`Queue1Protected`),
  - `private` (`Queue1Private`).

---

## 1) Соответствие требованиям задания

Ниже — как именно проект закрывает пункты из условия.

1. **Разделение на `.h` и `.cpp` + отдельный `main.cpp`**
   - `Queue.h` / `Queue.cpp` — базовый класс.
   - `Queue1Public.h` / `Queue1Public.cpp`.
   - `Queue1Protected.h` / `Queue1Protected.cpp`.
   - `Queue1Private.h` / `Queue1Private.cpp`.
   - `main.cpp` — меню и запуск программы.

2. **Элемент очереди содержит `int` и ссылку на предыдущий элемент**
   - `struct Node { int data; Node* prev; }` внутри `Queue`.

3. **Динамическая память и корректное удаление**
   - Добавление: `new Node(...)` в `enqueue`.
   - Удаление: `delete` в `dequeue` и очистка в `clear`.
   - В деструкторе вызывается `clear`, утечки не остаются.

4. **Количество элементов заранее неизвестно**
   - Элементы вводятся пользователем через меню и добавляются постепенно.

5. **Без STL-контейнеров и без статических массивов**
   - Нет `std::vector`, `std::list`, массивов фиксированного размера и т.п.

6. **Данные базового класса — в `private`**
   - `tail`, `size`, `Node` и вспомогательные методы находятся в закрытой части `Queue`.

7. **Меню 1–7**
   - Реализованы пункты:
     1) добавить,
     2) извлечь,
     3) вывести,
     4) вычислить требуемое значение,
     5) создать копию,
     6) слить с копией,
     7) выход.

8. **Три режима наследования**
   - Реализованы и доступны из главного меню:
     - `public`,
     - `protected`,
     - `private`.

## 2) Что именно делает программа

### Базовый класс `Queue`

Поддерживает:
- `enqueue(int)` — добавить элемент в конец очереди,
- `dequeue(int&)` — извлечь **первый** элемент очереди,
- `print()` — вывести очередь в порядке FIFO,
- `createCopy()` — создать глубокую копию,
- `merge(const Queue&, const Queue&)` — слить две очереди в новую, сохраняя порядок,
- `clear()` — очистить очередь,
- `isEmpty()`, `getSize()`.

### Дочерние классы `Queue1*`

Во всех трёх классах реализованы:
- `findRequiredValue()`
- `printRequiredValue()`

В текущем варианте "требуемое значение" = **количество положительных элементов**.

> Если в вашей таблице вариантов в `L1.pdf` требуется другое значение,
> замените логику в `findRequiredValue()` в файлах:
> - `Queue1Public.cpp`
> - `Queue1Protected.cpp`
> - `Queue1Private.cpp`

## 5) Как объяснять на защите (готовый сценарий)

Ниже можно говорить практически дословно.

### 5.1. Почему это очередь FIFO

- В структуре хранится цепочка узлов, где каждый узел знает "предыдущий" (`prev`).
- `tail` указывает на самый новый элемент.
- При печати и копировании используется обход от самого старого к самому новому.
- При извлечении удаляется именно самый старый элемент, поэтому поведение FIFO сохраняется.

### 5.2. Почему нет утечек памяти

- Каждый `enqueue` делает `new`.
- Каждый извлеченный элемент удаляется через `delete`.
- При завершении объекта деструктор вызывает `clear()`, удаляющий оставшиеся узлы.

### 5.3. Как сделаны копирование и слияние

- `createCopy()` создает **новую** очередь и добавляет в нее копии значений исходной очереди
  в том же порядке.
- `merge(q1, q2)` создает третью очередь: сначала все элементы `q1`, потом `q2`.
  Исходные очереди не портятся.

### 5.4. Чем отличаются public/protected/private наследование

- `Queue1Public : public Queue`
  - публичные методы `Queue` остаются публичными в `Queue1Public`.
- `Queue1Protected : protected Queue`
  - публичные/защищенные члены `Queue` становятся защищенными,
  поэтому снаружи доступны только специальные обертки (`add`, `extract`, `printQueue` и т.д.).
- `Queue1Private : private Queue`
  - унаследованные члены становятся приватными,
  снаружи снова работают только явно предоставленные обертки.

Это и демонстрирует три требуемых режима доступа.
